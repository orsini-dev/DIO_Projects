1st Project - HTML Tag Sheet

Introduction

HTML stands for Hyper Text Markup Language and it is the standard markup language for creating webpages. In essence, an HTML file is a simple text file that, instead of the default.txt, uses the .html extension to inform the browser how to interpret the file. It defines the structure of a webpage by defining a series of elements, or tags, that tell the browser how to display the content stored in them.

Tags in HTML usually look like this: <tag>content</tag>

They consist of an opening <tag> and a closing </tag> (identified by the forward slash "/"), with the content to be displayed placed in between the two. Some tags do not require a closing tag, such as the <br> tag, which only defines a line break in content. Others accept some specification within the tag itself, which is where atributes are included (e.g. <strong style="color: blue;">this text is bold and blue</strong>).

Bellow is a list of some basic HTML tags used to structure a simple webpage.

Intro References

<!DOCTYPE>

All HTML documents must start with a <!DOCTYPE> declaration. It is the very first tag to appear on an HTML file and tells the browser what document type to expect.

Index

<html>

The <html> tag represents the root of an HTML document. It is the container for all other HTML elements (except for the <!DOCTYPE> tag). You should always include the lang attribute inside the <html> tag, to declare the language of the Web page. This is meant to assist search engines and browsers.

Index

<head>

The <head> element is a container for metadata (data about data) related to the HTML document. Metadata is not displayed by the browser and typically define the document title, character set, styles, scripts, and other meta information.

Index

<meta>

The <meta> tag defines metadata about an HTML document. <meta> tags always go inside the <head> tag, and are typically used to specify character set, page description, keywords, author of the document, and viewport settings. They will not be themselves displayed on the page are used by browsers (how to display content or reload page), search engines (keywords), and other web services.

Index

<title>

The <title> tag defines the title of the document. The title must be text-only, and it is shown in the browser's title bar or in the page's tab. The <title> tag is mandatory in HTML documents. Its contents are very important for search engine optimization (SEO), as they are used by algorithms to decide the order of pages listed in search results. Among other things, the <title> tag can define a title in the browser toolbar, provide a title for the page when added to favorites and display it in search-engine results.

Note: You cannot have more than one <title> tag in an HTML document.

Index

<body>

The <body> tag defines the document's body. It contains all the contents of an HTML document, such as headings, paragraphs, images, hyperlinks, tables, lists, etc.

Note: There can only be one <body> tag in an HTML document.

Index

<header>

The <header> tag represents a container for introductory content or a set of navigational links. It typically contains one or more heading elements (<h1> - <h6>), a logo or icon and authorship information.

Note: You can have several <header> tags in one HTML document. However, they cannot be placed within a <footer>, an <address> or another <header> tag.

Index

<footer>

The <footer> tag defines a footer for a document or section. It typically contains authorship information, copyright information, contact information, sitemap, back-to-top links and related documents.

Note: You can have several <footer> tags in one document.

Index

<hr>

The <hr> tag defines a thematic break in an HTML page (e.g. a shift of topic). It displays a graphic horizontal line in the page, most often used to separate content or define a change in an HTML page. The <hr> tag is an empty tag, which means that it has no closing tag.

Index

<br>

The <br> tag inserts a single line break and is useful for moving down a line without adding paragraph space, such as in addresses or poems. Like the <hr> tag, <br> is also an empty tag.

Index

<h1> - <h6>

The <h1> to <h6> tags are used to define HTML headings.

<h1> defines the most important heading.

<h6> defines the least important heading.

Note: Only one <h1> should be used per page - this should represent the main heading/subject for the whole page. Also, make sure not to skip heading levels - start with <h1>, then use <h2>, and so on.

Index

<em>

The <em> tag is used to define emphasized text. The content inside is typically displayed in italic. A screen reader will pronounce the words inside <em> with an emphasis, using verbal stress.

Index

<strong>

The <strong> tag is used to define text with strong importance and is typically displayed in bold. It should be used to specify bold text without any extra importance.

Index

<mark>

The <mark> tag defines highlighted text.

Index

<b>

The <b> tag specifies bold text without any extra importance. According to the HTML5 specification, the <b> tag should be used as a last resort, when no other tag is more appropriate. The specification states that headings should be denoted with the <h1> to <h6> tags, emphasized text should be denoted with the <em> tag, important text should be denoted with the <strong> tag, and marked/ highlighted text should be denoted with the <mark> tag.

Index

<i>

The <i> tag defines a part of text in an alternate tone and is typically displayed in italic. The <i> tag is often used to indicate a technical term, a phrase from another language, a thought, a ship name, etc. It should be udes only when there is no other more appropriate semantic element, such as: <em>, <strong> or <mark>.

Index

<u>

The <u> tag defines text that is underlined. In HTML 5 it typically represents text that is unarticulated or has a non-textual annotation. Underlining a portion of text for purely stylistic reasons should be achieved using CSS.

Index

<sub>

The <sub> tag defines subscript text, which is displayed half a character below the normal line and is sometimes rendered in a smaller font. Subscript text can be used for chemical formulas, like H2O.

Index

<sup>

The <sup> tag defines superscript text, which is displayed half a character above the normal line, and is sometimes rendered in a smaller font. Superscript text can be used for footnotes, like Reference[1], or mathematical power representations, like X2.

Index

<small>

The <small> tag defines smaller text, such as copyright and other side-comments.

Index

<section>

The <section> tag defines a section in a document. It is useful for separating parts of the page with different purposes, making it easier style broader portions of content.

Index

<article>

The <article> tag specifies independent, self-contained content. An article should make sense on its own and it should be possible to distribute it independently from the rest of the site. Potential sources for the <article> tag are forum posts, blog posts or news stories.

Index

<p>

The <p> tag defines a paragraph of related content, which could be text, image and other formats. Browsers automatically add a single blank line before and after each <p> tag.

Index

<ol>

The <ol> tag defines an ordered list, which can be numerical or alphabetical, such as:

  1. Item 1
  2. Item 2
  3. Item 3

Index

<ul>

The <ul> tag defines an unordered (bulleted) list, such as:

  • Item 1
  • Item 2
  • Item 3

Index

<li>

The <li> tag defines a list item and is used inside both ordered and unordered lists. In <ul> the list items will usually be displayed with bullet points. In <ol>, the list items will usually be displayed with numbers or letters.

Index

<blockquote>

The <blockquote> tag specifies a section that is quoted from another source. Browsers usually indent <blockquote> tags.

"This is a quoted text"

Index

<del>

The <del> tag defines text that has been deleted from a document, usually displayed with line striking it through.

Index

<s>

The <s> tag specifies rectified text which is no longer correct, accurate or relevant. Like the <del> it will be displayed with a line through it.

Index

<ins>

The <ins> tag defines a text that has been inserted into a document, usually displayed underlined.

Index

<abbr>

The <abbr> tag defines an abbreviation or an acronym, like HTML.

Note: The global title attribute can be used to show the description for the abbreviation/acronym when you mouse over the element.

Index

<code>

The <code> tag is used to define a piece of computer code. The content inside is displayed in the browser's default monospace font, such as:

<p> This is a piece of HTML code. </p>

Index

<a>

The <a> tag defines a hyperlink, which is used to link one page to another or even diferent places inside the same page. The most important attribute of the <a> tag is the href attribute, which indicates the link's destination. If the <a> tag has no href attribute, it is only a placeholder for a hyperlink. A linked page is normally displayed in the current browser window unless specified by an attribute.
